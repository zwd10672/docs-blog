    02- Vue 项目中使用 mock 数据

    在 Vue 项目中使用 mock 数据，尤其是在 Vue Element Admin 这样的框架中，通常遵循以下步骤：

    # 1. 安装并配置 mock 工具

    首先，确保你的项目中已经安装了 mock 相关的库，如mockjs或直接使用 Vue Element Admin 提供的 mock 方案。如果未安装，可以通过 npm 或 yarn 安装：

    npm install mockjs --save-dev
    1

    # 2. 创建 mock 文件

    在项目根目录下或特定的 mock 文件夹内创建.js文件，用于定义 mock 规则。例如，创建一个mock/index.js，并在其中编写模拟接口的逻辑。以下是一个简单的示例，模拟一个用户登录接口：

    import Mock from "mockjs"

Mock.mock("/api/login", "post", options => {
  const { username, password } = JSON.parse(options.body)
  // 假设用户名为'admin'，密码为'password'时登录成功
  if (username === "admin" && password === "password") {
    return {
      code: 200,
      message: "登录成功",
      token: "your-token-here"
    }
  } else {
    return {
      code: 401,
      message: "用户名或密码错误"
    }
  }
})

    # 3. 配置 Vue 项目以使用 mock

    在 Vue 项目的配置文件（如vue.config.js或特定框架的配置文件）中，配置 mock 服务启动。Vue CLI 3 及以上版本中，可以这样配置：

    module.exports = {
  devServer: {
    before: require("./mock/index.js")
  }
}

    这段代码会在开发服务器启动前执行 mock 文件中的逻辑，拦截对应的 API 请求并返回模拟数据。

    # 4. 在前端代码中发起请求

    使用 axios 或其他 HTTP 客户端库发起请求，就像请求真实后端 API 一样。例如：

    import axios from "axios"

axios
  .post("/api/login", {
    username: "admin",
    password: "password"
  })
  .then(response => {
    console.log(response.data)
  })
  .catch(error => {
    console.error("登录失败:", error)
  })

    # 5. 切换到真实 API

    当准备将应用部署到生产环境或需要测试真实 API 时，确保关闭或禁用 mock 逻辑。这通常通过环境变量或配置文件来控制，具体方法取决于你的项目配置。

    # 注意

    • 确保在开发环境下使用 mock 数据，避免在生产环境中暴露 mock 逻辑。
    • 根据项目需求，可以灵活配置哪些接口使用 mock，哪些直接调用真实 API。
    • 保持 mock 数据与实际后端 API 返回格式一致，以便前端组件能够无缝对接。
